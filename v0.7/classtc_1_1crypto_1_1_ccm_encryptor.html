Detailed Description

template<class BlockCipher>
class tc::crypto::CcmEncryptor< BlockCipher >

Class for CCM mode encryption/decryption.

Template Parameters
BlockCipherThe class that implements the block cipher used for CCM mode authenticated encryption/decryption.

Counter with CBC-MAC (CCM) mode is an authenticated block cipher. It is designed to be used with small discrete payloads, encrypted/decrypted in one go.

This class is a template class that takes a block cipher implementation class as template parameter. See Aes128CcmEncryptor or similar for supplied realizations of this template class.

The implementation of BlockCipher must satisfies the following conditions. See AesEncryptor or similar class, for more information including parameters to each function.

  1. Has a kBlockSize constant that defines the size of the block to process.
  2. Has a kKeySize constant that defines the required key size to initialize the block cipher.
  3. Has an initialize method that initializes the state of the block cipher.
  4. Has an encrypt method that encrypts a block of input data.
  5. Has a decrypt method that decrypts a block of input data.

This class has two states:

  • None : Not ready
  • Initialized : Ready to process data

General usage of this class is as follows:

  • Initialize CCM state with initialize().
  • Encrypt or decrypt data using encrypt() or decrypt().
    • When using decrypt() you must compare the generated MAC with the expected one, alternatively you can use decrypt_and_verify()

Parameters
[out]dstBuffer where decrypted data will be written.
[in]srcPointer to data to decrypt.
[in]sizeSize in bytes of data to encrypt.
[in]ivPointer to initialization vector.
[in]iv_sizeSize in bytes of initialization vector.
[in]addPointer to additional authentication data.
[in]add_sizeSize in bytes of additional authentication data.
[out]tagBuffer where message authentication code will be written.
[in]tag_sizeSize in bytes of message authentication code.
Precondition
  • size > 0. Maximum value for size depends on iv_size.
    • Where iv_size == 7 size <= 0xffffffffffffffff (64bits).
    • Where iv_size == 8 size <= 0xffffffffffffff (56bits).
    • Where iv_size == 9 size <= 0xffffffffffff (48bits).
    • Where iv_size == 10 size <= 0xffffffffff (40bits).
    • Where iv_size == 11 size <= 0xffffffff (32bits).
    • Where iv_size == 12 size <= 0xffffff (24bits).
    • Where iv_size == 13 size <= 0xffff (16bits).
  • tag_size is 4, 6, 8, 10, 12, 14, or 16.
  • iv_size is between 7 and 13 inclusive.
  • add_size is between 0 and 0xff00.
Postcondition
  • Decrypted data is written to dst.
  • Message authentication code is written to tag.

This decrypts/signs the data in src, writing it to dst, and writing message authentication code to tag.

Note
  • dst and src can be the same pointer.
Exceptions
