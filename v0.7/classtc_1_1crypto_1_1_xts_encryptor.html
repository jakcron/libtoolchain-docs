Detailed Description

template<class BlockCipher>
class tc::crypto::XtsEncryptor< BlockCipher >

Class for XTS mode encryption/decryption.

Template Parameters
BlockCipherThe class that implements the block cipher used for XTS mode encryption/decryption.

XTS (XEX mode with ciphertext stealing) mode is designed for disk encryption, and as such operates on sectors of data rather than blocks. Unlike XEX (XOR encrypt XOR) mode the sector size does not have to be a multiple of the block size.

This class is a template class that takes a block cipher implementation class as template parameter. See Aes128XtsEncryptor or similar for supplied realizations of this template class.

The implementation of BlockCipher must satisfies the following conditions. See AesEncryptor or similar class, for more information including parameters to each function.

  1. Has a kBlockSize constant that defines the size of the block to process.
  2. Has a kKeySize constant that defines the required key size to initialize the block cipher.
  3. Has an initialize method that initializes the state of the block cipher.
  4. Has an encrypt method that encrypts a block of input data.
  5. Has a decrypt method that decrypts a block of input data.

This class has two states:

  • None : Not ready
  • Initialized : Ready to process data

General usage of this class is as follows:

  • Initialize XTS state with initialize().
  • Encrypt or decrypt sector(s) using encrypt() or decrypt().

Initializes the XTS encryption state.

Parameters
[in]key1Pointer to key1 data.
[in]key1_sizeSize in bytes of key1 data.
[in]key2Pointer to key2 data.
[in]key2_sizeSize in bytes of key2 data.
[in]sector_sizeSize in bytes of the XTS data unit.
[in]tweak_word_orderBoolean to determine word order of tweak. True = LittleEndian, False = BigEndian.
Precondition
  • key1_size == kKeySize.
  • key2_size == kKeySize.
  • sector_size >= kBlockSize.
Postcondition
  • Instance is now in a Initialized state.

This resets the XTS state, initializes the two key schedules and the initialization vector (IV). The IV should be the IV for sector 0. Using encrypt() / decrypt() will update the IV as required.

Note
  • This must be called before performing encryption/decryption.
Exceptions
tc::ArgumentNullExceptionkey1 was null.
tc::ArgumentOutOfRangeExceptionkey1_size did not equal kKeySize.
tc::ArgumentNullExceptionkey2 was null.
tc::ArgumentOutOfRangeExceptionkey2_size did not equal kKeySize.
tc::ArgumentOutOfRangeExceptionsector_size was less than kBlockSize.
