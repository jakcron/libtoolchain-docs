Detailed Description

template<size_t KeyBitSize, class HashFunction>
class tc::crypto::RsaOaepEncryptor< KeyBitSize, HashFunction >

Class for RSA-OAEP encryption/decryption.

Template Parameters
KeyBitSizeRSA key size in bits.
HashFunctionThe class that implements the hash function used with RSA-OAEP.

This class is a template class that takes a key size and a hash function implementation class as template parameter. See Rsa2048OaepSha2256Encryptor or similar for supplied realizations of this template class.

The KeyBitSize is the size in bits of the RSA key, this only supports key sizes aligned to 8 bits.

The implementation of HashFunction must satisfies the following conditions. See Sha2256Generator or similar class, for more information including parameters to each function.

  1. Has a kBlockSize constant that defines the size of the block to process.
  2. Has a kHashSize constant that defines the output size of the hash value.
  3. Has an initialize method that begins processing.
  4. Has an update method that updates the hash value on input.
  5. Has a getHash method that gets the final hash value.

This class has two states:

  • None : Not ready
  • Initialized : Ready to process input data

General usage of this class is as follows:

  • Initialize RSA-OAEP Encryptor state with initialize().
  • Encrypt/Decrypt message with encrypt() / decrypt().

Resets the RSA calculation state with an RSA key. OAEP encoding uses a label (which is digested using a hash algorithm) as part of the MGF1 masking process. It is possible to specify a pre-digested label instead, in which case set isLabelDigested to true.
