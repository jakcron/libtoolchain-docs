Detailed Description

template<class HashFunction>
class tc::crypto::Pbkdf2KeyDeriver< HashFunction >

Class for deriving a key using Password-Based Key Derivation Function 2 (PBKDF2).

Template Parameters
HashFunctionThe class that implements the hash function used for key derivation.

PBKDF2 is a hmac based key derivation function, as defined in RFC 8018. As such this template class requires HashFunction to implement one of the following hash functions to be compliant with RFC 8018.

  1. SHA-1 (see Sha1Generator)
  2. SHA-224
  3. SHA-256 (see Sha256Generator)
  4. SHA-384
  5. SHA-512 (see Sha512Generator)

The implementation of HashFunction must satisfies the following conditions. See Sha256Generator or similar class, for more information including parameters to each function.

  1. Has a kBlockSize constant that defines the size of the block to process.
  2. Has a kHashSize constant that defines the output size of the hash value.
  3. Has an initialize method that begins processing.
  4. Has an update method that updates the hash value on input.
  5. Has a getHash method that gets the final hash value.

This class has three states:

  • None : Not ready
  • Initialized : Ready to derive key data

General usage of this class is as follows:

  • Initialize PBKDF2 calculation with initialize().
  • Derive key data with getBytes().

Parameters
[out]keyPointer to the buffer storing the derived key.
[in]key_sizeSize of key to derive.
Precondition
  • Instance is in an Initialized state.
Postcondition
  • The derived key is written to key.
Exceptions
tc::crypto::CryptoExceptionkey_size was too large.
Note
  • This method can be called successively to continue deriving key data for up to kMaxDerivableSize bytes.
  • If the instance is in a None state, then this call does nothing.
